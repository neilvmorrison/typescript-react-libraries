# try_catch

Wrapped try-catch that returns a Result type instead of throwing.

## Usage

```typescript
import { try_catch } from '@libraries/utilities';

const result = await try_catch(async () => {
  const response = await fetch('/api/users');
  return response.json();
});

if (result.error) {
  console.error('Error:', result.error);
} else {
  console.log('Data:', result.data);
}
```

## API Reference

### Import

```typescript
import { try_catch } from '@libraries/utilities';
```

### Parameters

| Name | Type               | Required | Description                   | Default |
| ---- | ------------------ | -------- | ----------------------------- | ------- |
| `fn` | `() => Promise<T>` | Yes      | The async function to execute | N/A     |

### Returns

| Name     | Type            | Description                                                        |
| -------- | --------------- | ------------------------------------------------------------------ |
| (result) | `IResult<T, E>` | A promise that resolves to either a success or error result object |

### Result Structure

**Success:**

```typescript
{
  data: T;
  error: null;
}
```

**Error:**

```typescript
{
  data: null;
  error: E;
}
```

## Benefits

- No need to wrap code in try-catch blocks
- Type-safe error handling
- Works with both sync and async functions
- Cleaner error handling flow
